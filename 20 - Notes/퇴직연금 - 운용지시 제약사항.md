# 퇴직연금 - 운용지시 제약사항

> 주제 허브: [[퇴직연금]]
> 관련 노트: [[퇴직연금 - 자산운용 업무흐름]] · [[퇴직연금 - 매수매도 시스템 업무흐름]]

## Summary
퇴직연금 가입자(또는 사용자)의 운용지시는 법령·상품·시스템 측면에서 다양한 제약이 존재한다.
운용관리기관은 지시 접수 시 아래 제약 요건을 검토한 뒤 주문을 실행한다.

---

## 1. 법령상 제한 (근로자퇴직급여 보장법 제21조)

| 제한 항목 | 내용 |
|-----------|------|
| 위험자산 편입 한도 | 주식형 펀드 등 원리금 비보장 자산은 적립금의 **70% 이하** |
| 집중투자 한도 | 동일 발행자 증권은 적립금의 **10% 이하** / 동일 집합투자업자 펀드는 **30% 이하** |
| 주식 직접투자 금지 | 개별 주식 직접 매수 불가 — 펀드·ETF를 통한 간접투자만 허용 |
| 이해충돌 방지 | 퇴직연금사업자가 발행한 주식·후순위채권 편입 금지 |
| 대출·담보 금지 | 적립금으로 대출 실행 또는 담보 제공 불가 |
| 파생상품 제한 | 헤지 목적 외 파생상품 단독 편입 불가 |

---

## 2. 상품 적합성 제한

- **투자성향 적합성**: 가입자 투자성향(안정형·안정추구형·위험중립형 등) 검사 결과와 상품 위험등급 불일치 시 경고 팝업 → 가입자 확인 후 진행 가능 (강제 차단은 아님)
- **원리금보장형 의무 편입**: DC형·IRP는 일정 비율 이상 원리금보장형 상품(정기예금, GIC 등) 유지 권고
- **디폴트옵션 충돌**: 사전지정운용방법(디폴트옵션)이 지정된 경우, 별도 운용지시 없이 6주 경과 시 자동 운용 → 이후 가입자 지시 우선

---

## 3. 시스템·운영 제한

| 제한 항목 | 내용 |
|-----------|------|
| 운용지시 가능 시간 | 영업일 기준 특정 시간 내 (예: 09:00~15:30) — 이후 접수 분은 익영업일 처리 |
| 최소 거래 금액 | 상품별 최소 매수 금액 존재 (펀드별 상이) |
| 동일 종목 한도 | 단일 펀드·상품 편입 한도 (적립금 대비 %) |
| 1일 지시 횟수 | 일부 상품은 당일 1회 운용지시만 허용 |
| 기준가 적용 기준 | 당일 오후 3시 이전 접수 → 당일 기준가, 이후 → 익영업일 기준가 |

---

## 4. 처리 흐름에서의 제약 검토 위치

```
운용지시 접수
    ↓
[제약 검토 단계] ← 여기서 아래를 순차 확인
  ① 법령 한도 초과 여부 (위험자산 70% 초과?)
  ② 집중투자 한도 초과 여부 (동일 발행자 10%, 동일 운용사 30%)
  ③ 금지 상품 여부 (사업자 발행 주식 등)
  ④ 투자성향 적합성
  ⑤ 운용지시 가능 시간 여부
  ⑥ 최소 금액·횟수 한도
    ↓
[통과] → 주문 실행
[차단] → 오류 반환 → 가입자 안내
```

> 시스템 처리 상세: [[퇴직연금 - 매수매도 시스템 업무흐름]] 5-2-1(실패) 분기 참고

## Key Points
- 위험자산 70% 한도가 핵심 법령 제한 — DC형·IRP 모두 동일 적용
- 투자성향 부적합은 강제 차단이 아닌 확인 절차로 가입자 책임 부여
- 기준가 적용 기준시각(컷오프 타임)을 지키지 않으면 예상치 못한 날의 기준가 적용 가능
- 디폴트옵션 도입(2023년~) 이후 운용지시 미제출자 처리 흐름이 추가됨

## My Thoughts
제약사항은 단순한 규제가 아니라 가입자 보호 장치라는 관점이 중요하다.
위험자산 70% 한도나 이해충돌 방지 규정은 사업자가 가입자를 불리하게 운용하지 못하도록 막는 장치이며,
투자성향 적합성 절차는 가입자 본인이 위험을 인지하고 책임지게 하는 구조다.
시스템 개발 관점에서는 이 제약들이 모두 "운용지시 접수 → 검토 → 주문 실행" 중
**검토 단계**에서 코드로 구현되어야 한다는 점이 핵심이다.
