# 퇴직연금 - 매수/매도 시스템 업무흐름

> 상위 개요: [[퇴직연금 - 자산운용 업무흐름]]
> 이 노트는 시스템 레벨의 세부 단계를 다룬다.

## Summary
퇴직연금 자산운용 업무흐름 중 매수/매도 주문이 시스템 내에서 처리되는 단계별 흐름.
프론트 등록 → 투자자등록기관 전달 → 운용오케스트라 처리 → 결제/정산 → 통지로 이어진다.

---

## 매수/매도 등록 흐름 (프론트)

```
1. 매수/매도 등록 생성
   가입자 운용지시 입력

2. 투자성향 등록 생성
   가입자 투자성향 확인 및 적합성 검토

3. 상품선택 및 한도 생성
   선택 가능 상품 목록 조회, 한도 체크

4. 상품이력 조회
   기존 보유 상품 이력 확인

5. 투자자등록기관 전달
   ├─ 5-1. 전달입력 (데이터 전송)
   └─ 5-2. 업무처리
         ├─ 5-2-1. 운용오케스트라 처리 [실패]
         │         → 오류 처리 및 재시도/알림
         └─ 5-2-2. 운용오케스트라 처리 [성공]
                   → 다음 단계로 진행

6. 매수/매도 주문 전달
   운용관리기관 → 펀드사/ETF/예금기관

7. 매수/매도 결제
   T+2 또는 익영업일 결제

8. 설정배당금 정산 완료
   설정(매수) 완료 후 기준가 반영

9. 설정/해지대상 관리
   포트폴리오 편입·편출 처리

10. 기타시스템 연계
    외부 시스템 통보
```

## 백오피스 매수/매도 처리 흐름

```
1. 매수 등록 접수
2. 투자금 운용 지시
3. 투자금 배당 처리
4. 상품이력 조회
5. 거래내역 업무처리
   ├─ 5-1. 운용오케스트라 처리
   └─ 5-2. 결과 반영
6. 환불/매도 처리
7. 결제 처리
8. 설정배당금 정산 완료
9. 설정/해지 관리 & 결산처리
10~. 기타 후속 처리
```

---

## GIC 관련 흐름

- **GIC (Guaranteed Investment Contract)**: 원리금보장형 상품의 일종
- 가입자 운용지시 → GIC 상품 매수 처리 별도 흐름
- 운용관리기관 → 자산관리기관 전달 → 정산/결산

---

## SMS/Email 통지 흐름

```
1. 운용현황 보고 대상 생성
2. 가입자/수급자 목록 조회
3. 정산 완료 대상 추출
4. 결제/정산 결과 집계
5. 운용내역 통지 데이터 생성
6. SMS 발송 (처리 결과 즉시 통보)
7. SMS 발송 확인
8. Email 발송 (상세 내역 첨부)
9. SMS/Email 발송 완료 처리
10. 결제/정산 완료 통지
11. 운용현황 정기 통지 (분기 의무)
```

---

## Key Points
- **운용오케스트라**: 매수/매도 주문을 여러 기관으로 분배·조율하는 중간 처리 시스템
- 처리 실패(5-2-1) 시 재시도 또는 담당자 알림 루틴이 별도로 존재
- 백오피스는 프론트 등록과 독립적으로 처리 흐름을 가짐 (검증/이중화 목적)
- SMS는 즉시 통보, Email은 상세 내역 포함 통보로 역할 분리
