# 퇴직연금 - 자산운용 업무흐름

## Summary
퇴직연금 시스템에서 자산운용은 제도 유형(DB/DC/IRP)에 따라 운용 책임자와 흐름이 달라진다.

## 제도 유형별 운용 책임

| 유형 | 운용 책임자 | 특징 |
|------|------------|------|
| DB형 (확정급여형) | 사용자(회사) | 회사가 적립금 운용, 운용손익 회사 부담 |
| DC형 (확정기여형) | 가입자(근로자) | 근로자가 직접 운용지시, 운용손익 근로자 부담 |
| IRP (개인형) | 가입자 본인 | DC형과 유사, 이직/퇴직 시 개인이 계속 운용 |

## 자산운용 업무흐름 (DC형 / IRP 기준)

```
① 운용지시 접수
   가입자 → 퇴직연금사업자(운용관리기관)에 매수/매도 지시

② 적합성 검토
   운용관리기관 → 지시 가능 상품 여부, 한도 초과 여부 확인

③ 주문 실행
   운용관리기관 → 펀드사/ETF/예금기관에 주문 전달

④ 결제 및 청산
   주문 체결 후 대금 결제 (T+2 또는 익영업일)

⑤ 포트폴리오 반영
   자산관리기관이 가입자 계좌에 편입/편출 처리

⑥ 기준가 평가
   매 영업일 보유 자산 기준가 적용 → 잔액 산출

⑦ 운용현황 보고
   분기 1회 이상 가입자에게 운용 결과 통지 (의무)
```

## 주요 기관 역할

- **운용관리기관**: 운용지시 접수, 주문 실행, 상품 제공
- **자산관리기관**: 적립금 보관, 자산 편입·편출, 잔액 관리
  - 두 기관은 동일 금융기관이 겸영하거나 분리 운영 가능

## DB형의 차이점
- 운용지시 주체: 사용자(회사 재무담당)
- 부채(퇴직급여충당금) 대비 자산 적립 비율 관리가 핵심
- ALM(자산부채관리) 관점에서 운용전략 수립

## Key Points
- DC/IRP는 가입자가 직접 운용 → 금융이해력이 수익률에 직결
- 운용관리기관과 자산관리기관의 역할 분리는 횡령/유용 방지 목적
- 기준가 평가는 매일, 보고는 분기 의무
